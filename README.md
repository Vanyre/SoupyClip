# 🍲 SoupyClip

> **终端批量混剪脚本 · 竖屏随机拼接 · 自动叠加水印与背景音乐**
>
> *A lightweight batch video splicer for macOS, powered by Python + FFmpeg*

## 📖 简介

`video_splicer.py` 是一个面向本地素材库的命令行批量混剪脚本。

它会从视频素材池中随机抽取多个片段，重新拼接为一条新的竖屏短视频，并在输出时自动：

- 拼接多个随机视频片段
- 统一适配到 `1080x1920`
- 覆盖一层 PNG 水印
- 混入一条背景音乐
- 直接导出成可用成片

这类脚本的价值，不在于“复杂剪辑能力”，而在于把原本重复、机械、费时间的批量出片工作自动化。  
如果你手里已经有一批视频素材、贴图和音频，希望快速产出多条风格一致但组合不同的成片，这个脚本就是为这种场景准备的。

## 🧠 它是怎么工作的

脚本每生成一条视频，都会走一遍完整的随机混剪流程。

### 1. 随机生成本条视频的参数

- 成片总时长会在 `17 / 18 / 19 / 20` 秒之间随机选择
- 每条视频会随机使用 `6-9` 个片段
- 每个片段最短 `1` 秒
- 这些片段时长会先被随机切分，再用于后续匹配素材

### 2. 智能匹配足够长的视频素材

脚本启动时会先读取所有源视频的时长，建立缓存。

随后，它会根据每个目标片段的长度，从素材池里筛选出“时长足够”的候选视频，并优先选择使用次数更少的素材，尽量避免总是重复使用同几段源片。

同一条成片中，不会重复使用同一个源视频。

### 3. 均衡选择水印与背景音乐

- 从 `source_layers/` 中选一张 `.png`
- 从 `source_audios/` 中选一条 `.aac`

这两类素材也会按照“少用优先”的原则轮换，不会无限偏向同一张图或同一条音频。

### 4. 一次性完成整条视频输出

脚本会把这些步骤尽量合并进一次 FFmpeg 输出流程中完成：

1. 截取视频片段
2. 缩放并补边到竖屏尺寸
3. 拼接多个片段
4. 叠加 PNG 水印
5. 混入背景音乐
6. 输出最终成片

这样做的好处是：

- 中间文件少
- 磁盘 I/O 更低
- 批量生成时更省事
- 终端里可以直接看到单条任务进度

## ✨ 核心特性

- `🎞️` 批量生成多条随机拼接成片
- `📱` 输出统一为竖屏 `1080x1920`
- `🧩` 自动从多个视频里截取不同片段再拼接
- `🖼️` 自动叠加 PNG 水印
- `🎵` 自动混入背景音乐并按视频长度结束
- `⚖️` 视频 / 水印 / 音频都会尽量均衡使用
- `📊` 终端展示总进度、单条进度、任务结算
- `🔁` 单次失败会自动重试，不轻易中断整轮任务

## 🛠️ 运行环境

推荐环境：

- macOS
- Python 3.10+
- FFmpeg / ffprobe 已安装并可在终端直接调用

脚本默认使用 Apple 平台常见的硬件编码器：

```bash
h264_videotoolbox
```

所以它最适合在 Mac 上直接跑。如果你打算迁移到 Windows 或 Linux，通常需要自行修改编码参数。

## 📦 安装依赖

### 1. 安装 FFmpeg

如果你使用 Homebrew，可以直接执行：

```bash
brew install ffmpeg
```

安装后建议确认一下：

```bash
ffmpeg -version
ffprobe -version
```

### 2. 确认 Python 环境

```bash
python3 --version
```

这个脚本只依赖 Python 标准库，不需要额外安装第三方 Python 包。

## 📁 目录结构

脚本默认以 `video_splicer.py` 所在目录作为工程根目录，并使用以下固定结构：

```text
工程文件/
├── video_splicer.py
├── README.md
├── source_videos/
├── source_layers/
├── source_audios/
└── output_videos/
```

各目录用途如下：

- `source_videos/`
  放视频素材，当前识别 `.mp4`、`.mov`
- `source_layers/`
  放贴图 / 水印素材，当前识别 `.png`
- `source_audios/`
  放背景音乐，当前识别 `.aac`
- `output_videos/`
  存放生成好的成片

如果这些文件夹不存在，脚本首次运行时会自动创建。

## 🧱 素材准备建议

为了让脚本跑得更稳定，素材最好按下面的思路准备。

### 视频素材

- 数量尽量多一些
- 默认至少准备 `9` 个可用视频
- 每个视频建议明显长于 `1` 秒
- 素材越短，越容易在随机匹配时失败

### 水印素材

- 至少准备 `1` 张 `.png`
- 建议使用透明背景 PNG
- 若想铺满竖屏，最好提前按竖屏尺寸设计

### 音频素材

- 至少准备 `1` 条 `.aac`
- 时长长一些更稳妥
- 当前脚本默认读取 `.aac`，不是 `.mp3`

## 🚀 如何使用

### 方式一：直接运行，启动后交互输入数量

```bash
python3 video_splicer.py
```

如果你没有传 `--count`，脚本会先询问：

```text
这次要生成多少条？直接回车则使用默认 25 条:
```

直接回车，则默认生成 `25` 条。

### 方式二：命令行直接指定数量

```bash
python3 video_splicer.py --count 40
```

### 方式三：同时指定片段范围

```bash
python3 video_splicer.py --count 40 --min-clips 6 --max-clips 9
```

这适合你想更明确控制每条视频的节奏密度时使用。

## ⚙️ 参数说明

### `--count`

要生成的视频数量。

示例：

```bash
python3 video_splicer.py --count 20
```

### `--min-clips`

每条视频最少使用多少个片段，默认 `6`。

### `--max-clips`

每条视频最多使用多少个片段，默认 `9`。

示例：

```bash
python3 video_splicer.py --count 30 --min-clips 5 --max-clips 8
```

## 🏗️ 输出规则

所有成片都会直接输出到：

```text
output_videos/
```

文件名格式类似：

```text
20260318_201530_video_001.mp4
20260318_201530_video_002.mp4
20260318_201530_video_003.mp4
```

命名里包含：

- 本轮运行时间戳
- 本轮内的顺序编号

这样做的好处是：

- 不会轻易覆盖旧文件
- 不同批次容易区分
- 回头查找某次输出更方便

## 🖥️ 运行时会看到什么

脚本运行时，终端会显示一套比较清晰的任务信息：

- 当前素材池数量
- 本次目标生成条数
- 当前是第几条视频
- 当前尝试次数
- 当前视频的时长 / 片段数 / 水印 / 音频
- 单条视频的动态进度条
- 单条成功后的输出路径
- 整轮结束后的耗时与素材使用摘要

如果某次随机组合刚好失败了，脚本不会直接整轮中断，而是会自动回滚本次使用计数，并继续尝试下一组随机方案。

## ⚠️ 常见问题

### 1. 提示找不到 `ffmpeg` 或 `ffprobe`

说明 FFmpeg 没有安装，或者终端环境变量没有配置好。

先执行：

```bash
ffmpeg -version
ffprobe -version
```

如果命令无法识别，请先安装 FFmpeg。

### 2. 提示视频素材不足

常见原因包括：

- `source_videos/` 里的视频数量不够
- 可正常读取时长的视频数量不够
- 某些素材过短，无法满足本轮随机时长

常见解决方式：

- 增加更多视频素材
- 补充更长的源视频
- 降低 `--max-clips`

### 3. 提示缺少 PNG 或 AAC 素材

请检查：

- `source_layers/` 中是否真的有 `.png`
- `source_audios/` 中是否真的有 `.aac`

当前脚本不会自动读取 `.mp3`，如果你放进去的是 MP3，默认是不会被识别的。

### 4. 为什么有时会重试

因为每条视频都是随机组合出来的。

如果某一轮里刚好出现了“需要较长片段，但当前可选素材都不够长”的情况，脚本就会放弃这一轮随机组合，重新再试一组参数。这是正常行为，不一定是报错。

## 📝 注意事项

- 输出方向固定为竖屏 `1080x1920`
- 视频素材当前默认识别 `.mp4`、`.mov`
- 水印素材当前默认识别 `.png`
- 音频素材当前默认识别 `.aac`
- 默认编码器为 `h264_videotoolbox`
- 如果素材池偏小、视频偏短，而片段数设得又高，失败概率会明显上升

## 🎯 适用场景

这个脚本更适合下面这些场景：

- 本地有一批现成素材，希望快速批量出片
- 需要把同一批素材随机重组为多条不同成片
- 想通过命令行连续执行，而不是点 GUI
- 剪辑流程相对固定，只需要拼接、水印、音频三件事

如果你需要的是更复杂的时间线控制、字幕系统、镜头级精修，那就更适合在这个脚本基础上继续扩展，而不是把它当成完整的视频编辑器。
